# Changelog

## Unreleased

- **Subtitles widget:** **Fix intermittent failures — four bugs fixed:**
  1. **Display thread race:** replaced per-phrase thread spawning with a single `display-worker` thread and queue — eliminates interleaved HTTP POSTs that caused the display to flicker between old and new text.
  2. **Calibration threshold cap:** added `SUBTITLE_MAX_THRESHOLD` (default 200) so a noisy calibration window can no longer push the speech threshold above reachable levels, which previously caused "works one run, silent the next."
  3. **Stream overflow handling:** `_read_mono_from_stream` now returns the overflow flag; when the audio stream overflows mid-phrase (common during CPU-heavy Whisper inference), the corrupted phrase buffer is discarded instead of producing garbled transcriptions.
  4. **DELETE/POST race on first live draw:** added a 150 ms delay after the idle-screen DELETE before the first subtitle POST, preventing the device from processing them out of order and wiping the first subtitle.
  Also reduced consumer thread join timeout from 120 s to 10 s for faster shutdown.

- **Subtitles widget:** **Fix "nothing for ages, then everything at once":** silence detection failed in any environment with moderate ambient noise because `release_threshold` was computed without considering the noise p75, and a single noise spike above the threshold reset the entire silence counter to zero. Two fixes: (1) `release_threshold` now includes `p75_adj + 2` as a candidate so it sits above typical ambient noise; (2) a noise spike in the hysteresis zone now decays `silent_run` by 1 instead of resetting it to 0 — phrases now end via silence detection in ~2-3 s instead of stalling to the 45 s max-duration cap.

- **Subtitles widget:** **Terminal shows `Transcribed:` but strip is blank:** always log **`Idle screen POST OK`** after placeholder and **`First live subtitle frame POST OK`** after the first successful live draw (stderr) so we can tell HTTP succeeded vs never reached. New env: **`SUBTITLE_CLEAR_ON_EXIT=0`** skips **DELETE** on stop; **`SUBTITLE_SKIP_DELETE_BEFORE_LIVE=1`** skips **DELETE** before first live line (try if firmware mis-handles delete-then-draw).

- **Subtitles widget:** **Intermittent “no phrases” after restart:** calibration no longer uses **median** + raw **p75** only (talking or noise during “stay quiet” inflated thresholds). Uses **p20** for noise floor and caps **p75** contribution; warns on stderr if **p90** suggests loud sound during calibration.

- **Subtitles widget:** **Nothing on bar / ignored draws:** default live **`SUBTITLE_DRAW_PRIORITY`** is **9** (was hardcoded **7**) so subtitles win over more on-screen apps (BSB: lower priority is ignored while a higher-priority draw is active). Startup banner prints **`display`** and **`draw_priority`**. **`_post_display_draw`** logs **`POST /api/display/draw`** failures with **status + body**; **`DELETE`** failures log to stderr instead of staying silent.

- **Subtitles widget:** Removed temporary NDJSON debug-session logging (`_dbg_log`) from **`subtitle_widget.py`** after verification.
