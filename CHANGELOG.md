# Changelog

## Unreleased

- **Buses widget (`Published/Widgets/Buses/buses_fake.py`):** Added an **offline fake** runner (no TfL/network) that reuses `buses.py` formatting and `/api/display/draw` — configurable mock routes and looping countdown — for when live timetables are unavailable. Run: `python buses_fake.py` from `Published/Widgets/Buses/`. Default mock shows **286** / **Sidcup, Queen Mary’s Hospital** on both rows (different countdown phases via `_CYCLE`).
- **Buses widget (`Published/Widgets/Buses/buses.py`):** Set TfL stop to **Greenwich Town Centre / Cutty Sark** ([`490007515D`](https://tfl.gov.uk/bus/stop/490007515D/greenwich-town-centre-cutty-sark), stand D), removed hardcoded second-row route/destination so both rows use live API data, show the **full** `destinationName` (e.g. “Sidcup, Queen Mary’s Hospital”) with marquee scrolling, and shortened minute labels (e.g. `12m`) to avoid right-edge clipping. Poll TfL every **30s** and **only POST `/api/display/draw` when visible text would change** (route, destination, or minute label) so the destination marquee is not reset on every poll.
- **Bus Timetables (`Unpublished/APPS/Bus Timetables`):** Renamed from Buses v2 JS; display name **Bus Timetables**; device folder **`/ext/apps/bus_timetables`**; custom launcher icons **`icon_front.bin`** / **`icon_back.bin`** (front generated from `Bus Stop Icon.png`, back generated by `build_bus_icons.py`). **OK** from a city screen → bus selection; **Start** only opens city from menu. Menu **y** 1/9. README and upload script updated.
- **Bus Timetables (`Unpublished/APPS/Bus Timetables`):** Fixed `icon_front.bin` generation to use the correct Busy Bar format (**LVGL I4 indexed-color 8x8**), encoding `Bus Stop Icon.png` with transparent -> index 0 and red/white -> palette indices (for 3x6 PNGs: placed top-left into the 8x8 grid).
- **Bus Timetables (`Unpublished/APPS/Bus Timetables`):** Fixed palette byte order for `icon_front.bin` (swapped to RGBA packing) so the bus-stop icon doesn’t render with “blue” instead of red in the apps chooser.
- **Bus Timetables (`Unpublished/APPS/Bus Timetables`):** Tidied the app folder to only keep upload/runtime assets plus the icon generation inputs/scripts (removed unused A–G letter PNGs, background PNGs, and layout JSON files).
- **Bus Timetables (`Unpublished/APPS/Bus Timetables`):** Shifted `Bus Stop Icon.png` encoding 2 pixels to the right when generating `icon_front.bin` to align in the app chooser.
- Added `download_busybar_apps.py` at repo root: recursively downloads `/ext/apps/` from the BUSY Bar via `/api/storage/list` and `/api/storage/read` (default host `http://10.0.4.20`, default output `~/Downloads/BusyBar-apps`).
- **Social Battery Level (Custom App):** Simplified to show battery images only (`N_Battery.png`) with instant level switching (no Battery/Social alternating, no transitions).
- Added `upload_to_device.py` helper for deploying this app (uploads `main.lua`, `app.json`, and PNG assets to `/ext/apps/social_battery_level`).
- Added `next_bus` Busy Bar custom app: all files in `Unpublished/next_bus/` — `main.lua` (device app), `server.py` (HTTP proxy to run on computer; Busy Bar is HTTP-only, TfL is HTTPS-only).
- **Snake (Custom App):** Fixed wrap-through-edge deaths by allowing movement into the tail cell when not eating, so the snake can “pop through” to the other side cleanly.
- **Internet Speed (Custom App):** Translated `Unpublished/Internet Speed/speed.py` to English, improved robustness (timeouts, clearer errors, and optional CLI configuration), and fixed HTTP endpoints to match the project BUSY Bar API.
- **Internet Speed (Custom App):** Fixed `draw_speeds()` request payload to use valid BUSY Bar text color format (`#RRGGBBAA`), resolving `400 Bad Request` responses when updating RX/TX text.
- **Internet Speed (Custom App):** Improved `draw_speeds()` warning output to include the server response body when BUSY Bar rejects a request.
- **Internet Speed (Custom App):** Lowered the RX/TX text positions by ~3 pixels for better vertical alignment with the label artwork.
- **Water Timer (Custom App):** New app in `Unpublished/Water Timer/` with photo start screen, 30-minute countdown, flashing “DRINK/REFILL” prompt, then fill-up animation with count-up timer; dial adjusts elapsed time to speed up/slow down.
- **Water Timer (Custom App):** Countdown cup-down now renders from 30 per-minute 16x16 PNG frames in `cup_going_down/` (instead of a long .anim); fill phase uses `cup_2.anim`.
